1 - Stworzenie nowego brancha dla konkretnej wersji
PEP 745 – Python 3.14 Release Schedule.
 - 3.14 development begins: Wednesday, 2024-05-08
 - W dniu rozpoczęcia developmentu nowej wersji (PEP 745, 8 maja 2024) ktoś robi po prostu:
     git checkout main
     git checkout -b 3.14

        Od tego momentu:
            Wszystkie nowe funkcje 3.14 są wprowadzane tylko w branchu 3.14.
            main może dalej przyjmować poprawki i przygotowywać kod do przyszłych wersji (np. 3.15).

     Branch 3.14 jest czasem nazywany „development branch” lub „feature branch” dla tej wersji.
