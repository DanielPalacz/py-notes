  • 002. Jak osadzać kod JavaScript na stronie (wykorzystując tag script)? Kiedy skrypt jest 'blokujący' i co to oznacza?
    • 002a. Funkcja document.write zastosowana w różnych trybach osadzania kodu JS nie zawsze działa, bo jej stosowanie jest dozwolone tylko w trakcie ładowania HTML. defer i async powodują, że skrypt wykonuje się po załadowaniu HTML, więc document.write() jest ignorowane. Zamiast document.write() najlepsza alternatywą jest użycie innerHTML lub appendChild()
    • 002b. Osadzanie kodu JS w atrybutach HTML jest niezalecane.
    • Odpowiedź

  • 003. Deklaracja zmiennych: let, const, var lub domyślna. Jak działa zakres blokowy?
    • 003a. Domyślnie (bez użycia: let, const, var) zmienne trafiają do zakresu globalnego - chyba, że używamy trybu ścisłego.
    • 003b. Czy w Python istnieje podobny do 'const' mechanizm wymuszania, by zmienna była stała?
    • 003c. W Pythonie bloki takie jak if, for czy while nie tworzą nowego zakresu zmiennej - a w JavaScript tak.
    • 003d. W Pythonie brak deklaracji działa bardziej intuicyjnie – przypisanie zmiennej automatycznie umieszcza ją w odpowiednim zakresie (lokalnym lub globalnym).
    • 003e. Tryb ścisły w JavaScript, to konfiguracja wprowadzająca bardziej rygorystyczne zasady programowania ("use strict";).
    • Odpowiedź

  • 004. Globalny kontekst w JavaScript (przeglądarka). Kontekst globalny a obiekt window - jaka jest relacja między nimi?
    • 005a. Globalny kontekst w JavaScript zależy od środowiska (przeglądarka, Node.js, Web Workers).
    • 005b. Wprowadzono 'globalThis', żeby to wszystko połączyć. Użycie 'globalThis' to teraz zalecany sposób na wymuszenie globalnego kontekstu - działa na wszystkich typach środowisk.
    • Odpowiedź

  • 005. Typy danych w JavaScript.
    • 005a. Typy prymitywne, jest ich 7, nie mają prototypu i nie dziedziczą po Object.
    • 005b. JS ma 1 główny typ referencyjny: Object. Poza typami prymitywnymi, prawie wszystko w JS dziedziczy po Object.

AJAX:

Podstawy podstaw:
  • 201. Co to jest AJAX (Asynchronous JavaScript and XML)?
    • Odpowiedź
    - techniki programowania webowego, wykorzystujące różne technologie (przeglądarka) by tworzyć asynchroniczne aplikacje
- XMLHttpRequest (starsze API, skomplikowane w użyciu), fetch (bardziej nowoczesnym API, które ma na celu zastąpienie starszego)

Notatki:


2. Poznaj podstawy JavaScript
    Zacznij od prostych rzeczy:
        Co to jest JavaScript i do czego służy?
        Jak osadzać JS na stronie (np. script w HTML)?

    Naucz się podstawowych składników języka:
        Zmienne (let, const, var)
        Typy danych (string, number, boolean, array, object)
        Operatory (arytmetyczne, logiczne, porównania)
        Instrukcje warunkowe (if, else if, switch)
        Pętle (for, while, do...while)
        Funkcje (definiowanie, wywoływanie, funkcje strzałkowe)


3. Narzędzia i środowisko pracy
    Edytor kodu: Pobierz edytor jak Visual Studio Code (VS Code).
    Konsola w przeglądarce: Otwórz narzędzie programistyczne w swojej przeglądarce (zwykle F12) i używaj zakładki "Console".
    Naucz się korzystać z debuggera.


4. Manipulacja DOM
Dowiedz się, jak JavaScript może wpływać na HTML i CSS:
    Wybieranie elementów: document.querySelector, getElementById
    Zmiana zawartości HTML: .innerHTML, .textContent
    Zmiana stylów CSS: .style
    Obsługa zdarzeń: addEventListener


5. Praktyka z projektami
Stwórz małe projekty, by poćwiczyć swoje umiejętności:
    Licznik czasu
    Prosty kalkulator
    Gra w zgadywanie liczby
    Lista zadań (To-Do List)


6. Zrozumienie zaawansowanych koncepcji
Gdy poczujesz się pewniej, zacznij uczyć się:
    Obiektowego programowania (OOP)
    Promisów i async/await
    API w przeglądarce (np. fetch, localStorage)
    Modułów JS (np. import, export)


7. Frameworki i biblioteki (opcjonalnie)
Gdy dobrze opanujesz podstawy, możesz zainteresować się narzędziami, które ułatwiają pracę z JS:
    React (do budowy interfejsów)
    Vue lub Angular (alternatywne frameworki)
    Node.js (JavaScript na backendzie)
